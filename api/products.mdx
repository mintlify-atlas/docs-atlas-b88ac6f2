All product endpoints require authentication via JWT token cookie.

## List Products

<api method="GET" endpoint="/api-productos/productos">
Retrieve a list of all products.
</api>

### Authentication

**Required**: JWT token via cookie  
**Role**: Any authenticated user

### Request Parameters

No parameters required.

### Response

<ResponseField name="ok" type="boolean">
  Success indicator
</ResponseField>

<ResponseField name="statusCode" type="number">
  HTTP status code (200)
</ResponseField>

<ResponseField name="mensaje" type="string">
  Empty string
</ResponseField>

<ResponseField name="data" type="array">
  Array of product objects
  
  <Expandable title="product object">
    <ResponseField name="id" type="number">
      Product ID
    </ResponseField>
    
    <ResponseField name="nombre" type="string">
      Product name
    </ResponseField>
    
    <ResponseField name="precio" type="number">
      Product price
    </ResponseField>
    
    <ResponseField name="stock" type="number">
      Available stock quantity
    </ResponseField>
    
    <ResponseField name="descripcion" type="string">
      Product description
    </ResponseField>
    
    <ResponseField name="categoria_id" type="number">
      Category ID (foreign key)
    </ResponseField>
    
    <ResponseField name="categoria_nombre" type="string">
      Category name (from join)
    </ResponseField>
    
    <ResponseField name="creado" type="string">
      Creation timestamp
    </ResponseField>
    
    <ResponseField name="actualizado" type="string">
      Last update timestamp
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

```bash
curl -X GET http://localhost:3000/api-productos/productos \
  -b cookies.txt
```

### Example Response

```json
{
  "ok": true,
  "statusCode": 200,
  "mensaje": "",
  "data": [
    {
      "id": 1,
      "nombre": "Laptop Dell XPS 15",
      "precio": 1299.99,
      "stock": 15,
      "descripcion": "High-performance laptop with Intel i7",
      "categoria_id": 1,
      "categoria_nombre": "Electrónica",
      "creado": "2026-01-15T10:30:00.000Z",
      "actualizado": "2026-02-20T14:45:00.000Z"
    }
  ]
}
```

### Error Responses

<ResponseField name="401 Unauthorized">
  Missing or invalid authentication token
</ResponseField>

<ResponseField name="500 Internal Server Error">
  Server error
  
  ```json
  {
    "ok": false,
    "statusCode": 500,
    "mensaje": "Error al listar los productos.",
    "detalles": "Error message"
  }
  ```
</ResponseField>

## Create Product

<api method="POST" endpoint="/api-productos/productos">
Create a new product.
</api>

### Authentication

**Required**: JWT token via cookie  
**Role**: Any authenticated user

### Request Body

<ParamField body="nombre" type="string" required>
  Product name
</ParamField>

<ParamField body="precio" type="number" required>
  Product price
</ParamField>

<ParamField body="stock" type="number" required>
  Initial stock quantity
</ParamField>

<ParamField body="descripcion" type="string" required>
  Product description
</ParamField>

<ParamField body="categoria_id" type="number" required>
  Category ID (must exist in Categorias table)
</ParamField>

### Response

<ResponseField name="ok" type="boolean">
  Success indicator
</ResponseField>

<ResponseField name="statusCode" type="number">
  HTTP status code (201)
</ResponseField>

<ResponseField name="mensaje" type="string">
  "Producto registrado exitosamente."
</ResponseField>

<ResponseField name="data" type="null">
  No data returned
</ResponseField>

### Example Request

```bash
curl -X POST http://localhost:3000/api-productos/productos \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{
    "nombre": "Laptop Dell XPS 15",
    "precio": 1299.99,
    "stock": 15,
    "descripcion": "High-performance laptop with Intel i7",
    "categoria_id": 1
  }'
```

### Example Response

```json
{
  "ok": true,
  "statusCode": 201,
  "mensaje": "Producto registrado exitosamente.",
  "data": null
}
```

### Error Responses

<ResponseField name="401 Unauthorized">
  Missing or invalid authentication token
</ResponseField>

<ResponseField name="500 Internal Server Error">
  Server error or foreign key constraint violation
  
  ```json
  {
    "ok": false,
    "statusCode": 500,
    "mensaje": "Error al registrar el producto.",
    "detalles": "Error message"
  }
  ```
</ResponseField>

## Notes

- All timestamps are automatically managed by the database
- The `creado` field is set to `CURRENT_TIMESTAMP` on creation
- The `actualizado` field is updated to `CURRENT_TIMESTAMP` on every update
- Product data is retrieved from the `ProductosView` database view which includes category information
- Category ID must reference an existing category in the Categorias table
