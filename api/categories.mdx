All category endpoints require authentication via JWT token cookie.

## List Categories

<api method="GET" endpoint="/api-productos/categorias">
Retrieve a list of all categories.
</api>

### Authentication

**Required**: JWT token via cookie  
**Role**: Any authenticated user

### Request Parameters

No parameters required.

### Response

<ResponseField name="ok" type="boolean">
  Success indicator
</ResponseField>

<ResponseField name="statusCode" type="number">
  HTTP status code (200)
</ResponseField>

<ResponseField name="mensaje" type="string">
  Empty string
</ResponseField>

<ResponseField name="data" type="array">
  Array of category objects
  
  <Expandable title="category object">
    <ResponseField name="id" type="number">
      Category ID
    </ResponseField>
    
    <ResponseField name="nombre" type="string">
      Category name
    </ResponseField>
    
    <ResponseField name="icono" type="string">
      Icon identifier or class name
    </ResponseField>
    
    <ResponseField name="color" type="string">
      Color code (e.g., hex color)
    </ResponseField>
    
    <ResponseField name="descripcion" type="string">
      Category description
    </ResponseField>
    
    <ResponseField name="creado" type="string">
      Creation timestamp
    </ResponseField>
    
    <ResponseField name="actualizado" type="string">
      Last update timestamp
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

```bash
curl -X GET http://localhost:3000/api-productos/categorias \
  -b cookies.txt
```

    },
    {
      "id": 2,
      "nombre": "Ropa",
      "icono": "fa-shirt",
      "color": "#10B981",
      "descripcion": "Prendas de vestir y accesorios",
      "creado": "2026-01-10T08:05:00.000Z",
      "actualizado": "2026-01-10T08:05:00.000Z"

</ParamField>

## Notes

- All timestamps are automatically managed by the database
- The `creado` field is set to `CURRENT_TIMESTAMP` on creation
- The `actualizado` field is updated to `CURRENT_TIMESTAMP` on every update
- Category data is retrieved from the `CategoriasView` database view
- Deleting a category that has associated products will fail due to foreign key constraints
- Categories are returned ordered by name alphabetically
